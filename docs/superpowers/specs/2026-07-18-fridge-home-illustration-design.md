# 냉장고 일러스트 홈 화면 디자인 스펙

## 1) 목표
- 첫 화면의 빈 느낌을 줄이고, **냉장고 컨셉이 즉시 전달되는 홈 화면**으로 개선한다.
- 기존 사용자 동선(재료 관리 / 레시피 추천)은 유지하고, **시각 디자인만 강화**한다.

## 2) 범위
### 포함
- `frontend/src/components/HomeScreen.tsx` 레이아웃 리디자인
- `frontend/src/App.css` 홈 화면 전용 스타일 추가/개선
- `frontend/src/App.test.tsx` 홈 화면 텍스트/구조 관련 테스트 기대값 정리

### 제외
- 라우팅/상태관리/비즈니스 로직 변경
- 애니메이션 추가
- 백엔드/API 변경

## 3) 디자인 방향
- 톤앤매너: **일러스트 느낌(캐주얼/귀여움)**, 정적 화면
- 색상: 파스텔 기반(민트/하늘/크림), 가독성 우선
- 구성:
  1. 상단 Hero 영역
     - 냉장고 일러스트(이모지/간단한 도형 조합 가능)
     - 환영 타이틀 + 서브카피
  2. 하단 액션 카드 2개
     - 기존 기능 유지(재료 관리, 레시피 추천)
     - 카드 비주얼 강화(아이콘, 배지, 보조 설명)

## 4) 컴포넌트/파일 설계
### `HomeScreen.tsx`
- 구조를 다음 블록으로 재구성:
  - `home-hero` (일러스트 + 텍스트)
  - `home-actions` (기존 카드 2개 유지)
- 버튼 onClick 핸들러는 기존 props(`onManageIngredients`, `onOpenRecipes`) 그대로 사용

### `App.css`
- 홈 전용 스타일 클래스 추가:
  - `.home-hero`, `.home-hero-visual`, `.home-hero-content`
  - 카드 강화 스타일(배지/아이콘/간격/그림자)
- 반응형 기준:
  - 모바일에서 hero 영역은 세로 정렬
  - 데스크톱에서 hero 시각 요소와 텍스트가 균형 배치

## 5) 데이터 흐름/동작
- 데이터 흐름 변경 없음
- HomeScreen 렌더링 → 버튼 클릭 시 기존 화면 전환 흐름 유지

## 6) 에러 처리
- 신규 비동기 로직 없음
- 기존 이벤트 핸들러 재사용으로 추가 예외 처리 불필요

## 7) 테스트 전략
- 최소 범위 테스트 원칙:
  - 홈 화면 핵심 문구/버튼 노출 확인
  - 기존 버튼 클릭 기반 화면 이동 회귀 확인(기존 테스트 흐름 유지)
- 스타일 자체는 스냅샷 대신 텍스트/구조 단언 위주로 유지

## 8) 수용 기준(완료 조건)
1. 첫 화면에서 냉장고 컨셉이 즉시 인지된다.
2. 기존 동선(재료 관리/레시피 추천)은 변하지 않는다.
3. 화면이 이전보다 덜 비어 보이고, 카드/히어로 시각 계층이 명확하다.
4. 프론트 빌드가 성공한다.
